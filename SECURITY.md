# Security Policy

## Supported Versions

We only provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a potential security vulnerability in this project, please notify me by sending an email to **your-email**

### Please include:
* A description of the vulnerability.
* Steps to reproduce the issue (proof of concept).
* The potential impact if exploited.

### What to expect:
* I will acknowledge receipt of your report within 3 business days.
* I will provide an estimated timeframe for a fix.
* Once the issue is resolved, a public disclosure will be made, and you will be credited for the discovery (unless you prefer to remain anonymous).

## Scope
This policy applies only to the source code within this repository. We do not accept reports for:
* Vulnerabilities in third-party dependencies (please report those to the respective maintainers).
* Non-exploitable "best practice" suggestions.

Thank you for helping keep this project secure!
